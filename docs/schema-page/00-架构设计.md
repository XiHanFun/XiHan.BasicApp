## 2. 核心设计：Schema 单一事实源

### 2.1 分层

```
PageSchema（单一事实源，每页一份）
   │  pageCode / resourceCode / permissions / resource / fields[] / actions[] / views[]
   ├─ 派生 → 搜索表单（searchable 字段，经搜索设置重排/显隐）
   ├─ 派生 → 高级搜索（advancedSearch 字段，卡片内浮层）
   ├─ 派生 → 表格列（visible 字段，权限过滤 + 列设置覆盖显隐/顺序/固定/列宽）
   ├─ 派生 → 导出字段（exportable 字段）
   ├─ 派生 → 导入模板与校验（importable 字段）
   ├─ 派生 → 行速览卡（全部可读字段，含表格隐藏列）
   └─ 派生 → 字典依赖（dictionaryCode 收集 → 批量异步取值注入 options）
```

**禁止**在页面里再写 SearchSchema/TableSchema/ExportSchema —— 全部由 `PageSchema.fields[]` 经 **selector（纯函数派生）** 计算得出。

### 2.2 核心类型（与 `schema/types.ts` 实现一致）

```ts
interface PageSchema<TRow> {
  pageCode: string                 // 页面唯一码，偏好/视图按此维度存储
  pageName: string                 // 页面名称
  permissions?: string[]           // 页面级权限码（SaasPermissionCodes）
  resourceCode?: string            // 后端资源码（FLS 脱敏规则匹配；缺省不拉取）
  resource: SchemaResource<TRow>   // 归一化资源：page（列表）/ tree（树形）/ remove
  fields: ListFieldSchema<TRow>[]  // 字段单一事实源
  actions?: ActionSchema<TRow>[]   // 页面级/行级/批量级操作
  views?: ViewSchema[]             // 预置视图（系统视图，scope 预留）
  rowKey?: string                  // 默认 basicId
  batchRemovable?: boolean         // 内置批量删除（依赖 resource.remove）
  scrollX?: number                 // 表格横向滚动宽度
  pageSize?: number                // 默认每页数量
  tree?: { childrenKey?: string, defaultExpandAll?: boolean }  // 树形模式（存在即启用）
}
```

- `ListFieldSchema`：**16 种 dataType**（string/text/number/boolean/enum/date/datetime/money/percent/tag/json/image/avatar/email/phone/url）+ 布尔开关（visible/searchable/advancedSearch/sortable/filterable/exportable/importable/editable/readonly/required/treeColumn）+ `permission`/`dictionaryCode`/`options`/`formatter`/`width`/`minWidth`/`fixed`/`order`/`defaultValue`/`render`。
- `ActionSchema`：page/row/batch 三作用域 + `permission`/`confirm`/`visible(row)`/`disabled(row)`。

派生 selector（`schema/selectors.ts`，纯函数可测试）：

```
toSearchFields(schema, can)    → 常用搜索字段
toAdvancedFields(schema, can)  → 高级搜索字段
toColumns(schema, can, opts)   → 表格列（权限过滤 + 列设置覆盖显隐/顺序/固定/列宽 + 树形列 + 可拖拽列宽）
toExportFields(schema, can)    → 导出字段
toSortParams(sorter)           → Naive 排序状态 → 归一化 sortField/sortOrder
```

## 3. 页面布局（固定三区）

```
┌─ SearchPanel ──────────────────────────┐  常用条件 + [查询][重置][高级搜索▾][搜索设置]
├─ ActionPanel ──────────────────────────┤  页面级操作按钮 | 右侧工具栏 [自定义槽][刷新][导出][列设置][全屏]
├─ TablePanel ───────────────────────────┤  NDataTable（服务端分页/排序/多选/序号/固定列/列宽拖拽/骨架/空态/树形）
└────────────────────────────────────────┘
   批量浮条位于表格页脚（已选 X 条 / 清空 / 批量删除 / 批量操作），不挤压表格空间
   高级搜索 = 搜索卡片内下拉浮层（不挤压表格，暗色模式取主题色）
   行悬停 450ms 浮出速览卡（Peek）：全字段详情含隐藏列，Teleport 至 body 自适应定位
   列表区为定高卡片（flex-height），滚动只发生在表格内部
   小屏（≤767px）：常用搜索区收起、操作按钮圆形右对齐、分页防溢出
```

页面自有弹窗/抽屉（表单、详情）放在 `SchemaPage` 默认插槽中，行操作经统一 `@action` 事件分发。

## 5. 偏好体系与跨端同步

```
解析优先级：用户设置（后端 SysUserSetting，localStorage 兜底） > 系统默认（PageSchema 内置）
```

- **存储模型**：后端 `SysUserSetting`（`UserId × Scene × SettingKey` 组合键）。Scene=Page 时 SettingKey 为 `pageCode`，SettingValue 为 JSON，按分区承载：`table`（列设置/密度/风格/多选/序号）、`search`（搜索字段设置）、`views`（个人视图）。后端不解释语义，前端负责版本兼容解析。
- **同步策略**（`useUserSettingSync`）：读取时后端优先、本地兜底；写入 600ms 防抖、尽力而为（失败静默不阻塞交互）；「保存」动作手动触发落库，灵动岛展示同步反馈。
- **同步开关**：应用设置中「同步搜索设置」「同步表格设置」可分区关闭（关闭后仅 localStorage）。
- **多级合并**（系统/租户/团队 scope）：`ViewSchema.scope` 与设计接口已预留，后端 `SysUserSetting` 当前仅个人维度，**暂缓**，待真实多租户运营需求出现再实施。

## 10. 实施纪事

### 10.1 S1 核心闭环

- 框架落地 `frontend/packages/components/schema/`：types/selectors/renderer/useSchemaTable + SchemaPage/SchemaSearchPanel/SchemaActionPanel/SchemaTablePanel，固定三区布局 + 行操作「更多」菜单 + 批量浮条 + 工具栏插槽 + 默认插槽（承载页面弹窗/抽屉）。
- 删除过渡脚手架 `packages/components/crud/`；首页 `setting/dict` 迁移：字段单一事实源派生搜索区与表格列，详情抽屉 + 主从子表 + 表单弹窗保留于默认插槽，自定义带色 NTag 经 `field.render` 保留。

### 10.2 S2 高级能力

- **列设置 + 密度**（`useTableSettings` + `SchemaTableSettings`）：显隐/排序/固定/密度，按 `pageCode` 持久化；字段增减（权限变更）自动合并恢复。

## 11. 评估与暂缓项

### 11.1 异步导出任务中心（评估完成 → 暂缓）

- 现状：同步 CSV 设 5000 行上限，已覆盖管理页常规数据量；日志类大表可先用服务端筛选收窄后导出。
- 评估结论（除非出现真实 >5000 行诉求，否则不启动）：
  1. **缺资源注册表**：`SysResource.ResourceCode` 是权限/FLS 维度的逻辑资源码，不含「资源 → 实体类型/查询服务」映射；通用导出执行器需新建注册表并逐资源登记查询委托（21+ 项持续维护成本）。
  2. **越权风险面大**：绕过各 QueryService 直接查实体表导出，会绕开其中的权限码校验、DataScope 过滤与投影裁剪；复刻这三层等于重写一遍读侧。
  3. **字段语义在前端**：导出列、字典 label 映射、formatter 均由前端 PageSchema 派生，服务端导出只能给原始列值，体验反而劣化。
- 设计草案（启动时再细化）：`SysExportTask`（任务实体）+ 后台执行器（资源注册表分发到既有 QueryService + `FieldSecurityService.ApplyAsync` 脱敏出口 + CSV 流式写出）→ 产物存 `SysFile`（预签名 URL 下载）→ 站内通知送达。

### 11.2 暂缓项

- 偏好/视图多级 scope（System/Tenant/Team）合并：接口已预留，待多租户运营需求。
- 框架文案 i18n key 化：与全站文案策略一并决策。
- `message/record` 等异构记录页迁移：待评估迁移收益。
