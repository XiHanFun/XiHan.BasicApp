# Schema 驱动页面框架 · 架构设计

> 版本：v1.1（S1 已落地，`pnpm type-check` 通过） · 日期：2026-05-31
> 目标：在现有 `frontend/packages` 基础设施之上，沉淀一套 **Schema 驱动 / 配置驱动 / 权限感知 / 租户感知 / 偏好感知** 的企业级列表页框架，消除各页面重复的 Search/Table/Export schema。
> 落地方式：**集成进现有 packages 演进**（不另起炉灶）。
> 首期范围：**核心闭环 + 1 个真实页跑通**。

> 🟢 **S1 实施进度（已完成并提交，`commit a6e80ac9`）**
> - 框架落地于 `frontend/packages/components/schema/`，经 `~/components` 统一导出：
>   - `types.ts`：`PageSchema` / `ListFieldSchema`（16 dataType + 12 布尔开关 + permission + dictionaryCode + options + formatter）/ `ActionSchema`（page/row/batch 作用域）/ `ViewSchema` / `SchemaQueryParams` / `SchemaResource` / `SchemaSelectOption`
>   - `selectors.ts`：`toSearchFields` / `toAdvancedFields` / `toColumns` / `toExportFields` / `toSortParams`（纯函数派生，权限过滤 + order 排序）
>   - `renderer.ts`：`dataType → 单元格渲染` + 脱敏 formatter（maskPhone/maskEmail/maskIdCard）
>   - `useSchemaTable.ts`：服务端分页/搜索/排序/删除状态机
>   - `SchemaPage` / `SchemaSearchPanel` / `SchemaActionPanel` / `SchemaTablePanel`：固定三区布局 + 行操作「更多」菜单 + 批量浮条 + 工具栏插槽 + 默认插槽（承载页面自有弹窗/抽屉）
> - **已删除过渡脚手架** `packages/components/crud/`（及空目录 `query-builder/`）；`components/index.ts` 改为导出 `schema`。
> - 唯一使用 crud 的页面 `views/platform/dict/index.vue` 已迁移至 `<SchemaPage :schema>`：字段为单一事实源派生搜索区与表格列；详情抽屉 + 主从子表 + 两个表单弹窗保留于默认插槽；行操作经统一 `@action` 分发；自定义带色 NTag 列通过 `field.render` 保留语义色。
> - 权限：页面/字段/操作的 `permission` 经现有 `usePermission().hasPermission` 过滤。
>
> **S1 取舍 / 待 S2**：高级搜索 Drawer、列设置（显隐/拖拽/固定/宽度）、视图管理、批量动作业务实现、导出/导入、字典 `dictionaryCode` 异步取值、字段脱敏对接后端 FLS。

## 1. 现状基线（逐文件核对，决定"复用 vs 新建"）

### 1.1 可直接复用（已存在，质量达标）

| 能力 | 现有实现 | 复用方式 |
| --- | --- | --- |
| 权限判断 | `stores/user.ts` `hasPermission(code)`（含 `*` 超管）；`accessStore.accessCodes` | Schema 的 `permission` 字段 → `hasPermission` 过滤 |
| 权限组件/指令 | `components/rbac/AccessControl.vue`、`composables/useAccessDirective.ts`（`v-access`）、`hooks/usePermission.ts` | 按钮/字段级权限直接用 |

| 字典（动态） | 后端 `DictQueryService.GetDictItemPage/Tree`；前端 `api/modules/configuration/dict.ts` | `dictionaryCode` → 运行时拉取 |
| 字典（枚举元数据） | 后端 `EnumMetadataQueryService.GetAllEnums/GetEnum`；前端 `hooks/useEnumService` + `composables/useEnum` | 枚举型 `dictionaryCode` 走元数据 |
| 字典（静态兜底） | `constants/business.ts`（`STATUS_OPTIONS`/`GENDER_OPTIONS` 等 20+ 组） | 高频固定字典直接引用 |

| 资源 API 工厂 | `api/factory.ts` `defineResource`（query/command/page/detail/create/update/remove） | Schema 的数据源契约 |
| CRUD 骨架 | `components/crud/`（CrudPage/CrudTable/CrudSearchForm/useCrud） | 升级为 Schema 驱动的底座 |
| i18n | `packages/locales/langs/{zh-CN,en-US}/*.ts`（分模块） | 所有 title/label 走 i18n key |
| 列表查询契约 | 后端统一 `QueryConditions`（filters/sorts/keyword）+ `PageRequest` | Schema 字段 → QueryConditions 映射 |

### 1.2 缺失（后端无端点，需新增或前端降级）

| 规格要求能力 | 后端现状 | 设计决策 |
| --- | --- | --- |
| 用户偏好（density/pageSize/viewCode） | ❌ 无 `UserPreference` 表/端点 | **首期：前端 localStorage 持久化**（按 `pageCode` 维度）；二期补后端 `SysUserPreference` 实现跨端同步 |
| 视图管理（系统/租户/团队/个人视图） | ❌ 无 `TableView`/`PageView` | **首期：个人视图存 localStorage**；二期补后端 `SysPageView`（含 scope=System/Tenant/Team/Personal） |
| 列设置（显隐/排序/固定/宽度） | ❌ 无端点 | **首期：localStorage**（属于个人偏好子集） |
| 导出任务中心（异步导出） | ❌ 无 `ExportTask` | **首期：同步导出当前页/筛选结果**；异步任务中心二期（依赖后端任务系统，已有 `SysTask` 可复用） |
| 导入历史 | ❌ 无 `ImportHistory` | 二期 |
| 字段权限脱敏（手机/邮箱/身份证） | ⚠️ 后端有 `SysFieldLevelSecurity` 实体 | **首期：前端 formatter 兜底脱敏**；二期对接后端 FLS 下发脱敏规则 |
| 数据权限（数据范围） | ✅ 后端有 `DataScope`，查询自动注入 | 前端无需特殊处理，透明生效 |

> **重建约定一致**：与 refactor 系列一致，线上库完全重建，所有新端点按全新空库设计，不做存量兼容。

## 2. 核心设计：Schema 单一事实源

### 2.1 分层

```
PageSchema（单一事实源，每页一份）
   │  pageCode / permissions / fields[] / actions[] / views[]
   ├─ 派生 → 搜索表单（searchable 字段）
   ├─ 派生 → 高级搜索（advancedSearch 字段）
   ├─ 派生 → 表格列（visible 字段，权限过滤）
   ├─ 派生 → 导出字段（exportable 字段）
   ├─ 派生 → 导入模板（importable 字段）
   ├─ 派生 → 详情展示（所有可见字段）
   └─ 派生 → 表单字段（editable 字段）
```

**禁止**在页面里再写 SearchSchema/TableSchema/ExportSchema —— 全部由 `PageSchema.fields[]` 经 **selector（派生函数）** 计算得出。

### 2.2 核心类型（设计稿，实现见 types/）

```ts
// 字段类型见提示词 ListFieldSchema（dataType 16 种 / 12 个布尔开关 / permission / dictionaryCode / formatter ...）
interface PageSchema {
  pageCode: string                 // 页面唯一码，偏好/视图按此维度存储
  pageName: string                 // i18n key
  permissions: string[]            // 页面级权限码（SaasPermissionCodes）
  resource: CrudResource<...>      // 数据源（defineResource 产出）
  fields: ListFieldSchema[]        // 字段单一事实源
  actions: ActionSchema[]          // 页面级 + 行级操作
  views?: ViewSchema[]             // 预置视图（系统视图）
  rowKey?: string                  // 默认 basicId
}
```

派生 selector（纯函数，无副作用，可测试）：
```
toSearchFields(schema)      → 搜索表单项
toAdvancedFields(schema)    → 高级搜索项
toColumns(schema, ctx)      → 表格列（按 permission + 视图列状态过滤排序）
toExportFields(schema, ctx) → 导出字段
toImportTemplate(schema)    → 导入模板
toQueryDto(schema, form)    → 后端 QueryConditions（filters/sorts/keyword）
```

## 5. 配置优先级（偏好解析）

```
用户配置（localStorage，首期） > 团队配置 > 租户配置 > 系统配置（PageSchema 内置默认）
```
首期只实现「用户(localStorage) → 系统(schema 默认)」两级；中间两级二期补后端后接入，解析函数 `resolvePreference(pageCode)` 预留四级合并接口。

## 6. 与现有 CRUD 脚手架的关系

- `packages/components/crud`（K 任务）= **过渡底座**，保留不动；其 `useCrud` 被 `schema-page/useTable` 内部复用。
- `schema-page` = **Schema 驱动上层**；新页面用 `SchemaPage`，老页面（dict 等）维持 CrudPage，按需迁移。
- 不强制一次性迁移，避免大规模回归。

## 9. 设计原则校验（对照需求）

- ✅ Schema First：唯一 `PageSchema`，派生一切
- ✅ 配置驱动 / 高配置化：字段 12 开关 + dataType + dictionaryCode + permission
- ✅ 权限感知：字段/按钮/页面级权限走现有 `hasPermission`
- ✅ 租户感知：后端查询自动注入 TenantId/DataScope，前端透明；视图/偏好预留 scope
- ✅ 偏好感知：localStorage（首期）→ 四级合并（演进）
- ✅ 国际化：title/label = i18n key，禁止硬编码
- ✅ 服务端模式：分页/排序/筛选统一进 QueryConditions
- ✅ 长期演进：框架包独立、selector 纯函数可测、与现有设施零冲突
