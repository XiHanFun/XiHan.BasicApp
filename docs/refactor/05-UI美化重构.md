# 05 · UI 美化重构蓝图

> 上游：[00-总览](./00-总览.md) · 关联：[03-页面逻辑](./03-页面逻辑关系重构.md)
> 目标：把**超大单文件列表页**收敛为**统一 CRUD 脚手架 + schema 驱动**；令牌化视觉、保证亮/暗一致、统一加载/空/错误态。

---

## 1. 现状（实证）

- 技术栈：Vue 3.5 + Naive UI 2.44 + Tailwind 3（`darkMode:'class'`）+ TS + vue-i18n；monorepo `packages/*`（design/layouts/router/...）。
- 设计令牌：Tailwind 用 CSS 变量驱动，`primary: rgb(var(--primary) / <alpha-value>)`，与 Naive UI 主色同步；`@iconify/tailwind` 动态图标；`content` 扫描 `src` + `packages`。✅ 令牌基础已具备。
- 布局：`packages/layouts/basic`（侧栏/头部/标签栏/偏好抽屉等），暗色切换走 `class`。
- **核心问题**：列表页为巨型单文件，如 `views/system/user/index.vue` **1968 行**、`views/system/message/index.vue` 776 行，单文件内手搓 `NDataTable + NPagination + NModal + NDrawer + NForm + NTabs + 列定义 + 多状态`。每个页面各写一套搜索区、空态、加载态、暗色处理 → 风格漂移、维护成本高、复制扩散。

---

## 2. 目标：统一 CRUD 脚手架（最高收益）

抽取一套数据驱动的列表页组件，落在 `packages/components/`，让业务页只写**配置**不写**结构**。

### 2.1 组件分层
```
packages/components/crud/
  CrudPage.vue          列表页骨架：搜索区 + 工具栏 + 表格 + 分页 + 弹窗，编排全流程
  CrudSearchForm.vue    schema 驱动搜索表单
  CrudTable.vue         封装 NDataTable：加载/空/错误/选择/排序统一
  CrudFormModal.vue     schema 驱动新增/编辑（Modal 或 Drawer 可切）
  useCrud.ts            组合式：分页/查询/增删改/loading/选中 状态机，对接 defineResource
  types.ts              列 schema、表单 schema、操作按钮 类型
```

### 2.2 业务页 = 配置（目标形态）
```vue
<!-- views/system/user/index.vue（目标：从 1968 行 → 约 150 行配置） -->
<script setup lang="ts">
import { CrudPage } from '~/components/crud'
import { userApi } from '@/api'               // 来自 03 的 defineResource
import { useUserCrudSchema } from './schema'   // 列/表单/搜索 schema

const { columns, searchSchema, formSchema } = useUserCrudSchema()
</script>

<template>
  <CrudPage
    :resource="userApi"
    :columns="columns"
    :search-schema="searchSchema"
    :form-schema="formSchema"
    row-key="basicId"
  />
</template>
```
```ts
// views/system/user/schema.ts —— 纯声明
export function useUserCrudSchema() {
  return {
    searchSchema: [
      { field: 'keyword', label: '关键词', type: 'input' },
      { field: 'status', label: '状态', type: 'select', options: STATUS_OPTIONS },
    ],
    columns: [
      { key: 'userName', title: '用户名' },
      { key: 'nickName', title: '昵称' },
      { key: 'status', title: '状态', render: statusTag },
      { key: 'actions', title: '操作', actions: ['edit', 'delete'] },
    ],
    formSchema: [
      { field: 'userName', label: '用户名', type: 'input', rules: ['required'] },
      { field: 'email', label: '邮箱', type: 'input', rules: ['email'] },
    ],
  }
}
```

### 2.3 `useCrud` 状态机（对接资源工厂）
```ts
export function useCrud<T>(resource: Resource<T>, options) {
  const loading = ref(false), rows = ref<T[]>([]), total = ref(0)
  const query = reactive({ page: { pageIndex: 1, pageSize: 20 }, ...options.defaultQuery })
  async function load() {
    loading.value = true
    try { const r = await resource.page(query); rows.value = r.items; total.value = r.total }
    finally { loading.value = false }
  }
  // create / update / remove / 选中 / 排序 …… 统一封装
  return { loading, rows, total, query, load, /* ... */ }
}
```

> 复合页（用户管理含角色/部门 Tab）可在 `CrudPage` 基础上加插槽，或单独编排，但**表格/表单/状态仍复用脚手架原子组件**，不再各写一套。

## 3. 视觉令牌化与一致性

### 3.1 收敛设计令牌到单一来源
- 所有颜色/圆角/间距/阴影/层级走 CSS 变量（`packages/design`），亮/暗各一套值；组件**禁止**写死 `#hex`、魔法间距。
- 与 Naive UI 主题对接：用 `NConfigProvider` 的 `themeOverrides` 从同一组 CSS 变量派生，确保 Naive 组件与 Tailwind 工具类同源同色。
```ts
// packages/design/naive-theme.ts
export const themeOverrides = {
  common: {
    primaryColor: 'var(--n-primary)',
    borderRadius: 'var(--radius)',
    // …… 全部引用令牌
  },
}
```

### 3.2 暗色一致性
- 业务页**不出现**裸 `dark:` 散写颜色；统一用语义令牌（`bg-background`/`text-foreground`/`border-border`），令牌自身负责亮暗切换。
- 验收：每个重构页在亮/暗下各截图走查，对比度达 WCAG AA。

### 3.3 统一状态与微交互
- 加载：表格骨架屏 / 局部 spinner（统一组件，不各写）。
- 空态：统一 `NEmpty` 包装 + 文案/插画令牌。
- 错误态：统一错误占位 + 重试按钮。
- 操作反馈：增删改统一 toast/确认弹窗文案与位置。
- 密度：统一表格行高、卡片内距、表单标签宽度。

---

## 4. 迁移步骤（先沉淀再抽象，避免过早抽象）
1. 选 3 个代表页（如 `dict` 简单、`user` 复杂含 Tab、`log` 只读）实测共性。
2. 据共性抽 `CrudTable`/`CrudSearchForm`/`CrudFormModal`/`useCrud`，先在这 3 页落地。
3. 固化设计令牌 + Naive `themeOverrides` 单一来源；清理裸色值。
4. 验证亮/暗 + 各状态后，铺开其余列表页（platform/log/system）。
5. 复合页接入脚手架原子组件 + 插槽。

---

## 5. 验收
- [ ] 列表页平均体量大幅下降（目标单页 ≤ 约 200 行配置）。
- [ ] 表格/搜索/表单/状态全部走共享脚手架，无重复结构代码。
- [ ] 颜色/间距/圆角 100% 令牌化；无裸 hex、无散写 `dark:` 颜色。
- [ ] Naive 主题与 Tailwind 同源；亮/暗视觉一致（AA 对比度）。
- [ ] 加载/空/错误/反馈 状态统一。
